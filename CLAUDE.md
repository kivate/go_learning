# Go 語言學習文件撰寫規則

本目錄專門存放 Go 語言學習文件，目標讀者為**入門者**，所有文件使用**繁體中文**撰寫。

---

## 目標讀者

* 剛開始學習 Go 語言的開發者
* 有其他語言背景（Python、JavaScript、Java）但不熟悉 Go 特性的人
* 準備 Go 技術面試的工程師

撰寫時假設讀者不熟悉指標、記憶體管理、並發等概念，需要從基礎解釋。

---

## 文件結構規範

### 必要元素

每份文件需包含：

1. **標題與說明**：第一段說明這份文件涵蓋哪些內容、適合哪個學習階段
2. **章節編號**：使用 `## 1.`、`## 2.` 格式，子節使用 `### 1-1`
3. **程式碼範例**：每個概念至少一個完整可執行的範例
4. **圖解**：複雜機制（記憶體、Channel 流程）使用 ASCII 圖解說明
5. **總結對照表**：文件結尾提供重點摘要表格
6. **更新日期**：文件最後一行標示更新日期

### 章節命名慣例

```
## 1. 主題名稱
### 1-1 子主題
#### 具體說明項目
```

## Markdown 格式規則

以下規則避免 Markdown linting 警告：

### 標題

* **禁止**用 `**粗體**` 獨立成行當標題（MD036）—— 改用 `####` heading
* 標題前後必須有空白行（MD022）
* 標題文字不加句點

```markdown
<!-- 錯誤 -->
**解法一：shadow 變數**
說明文字

<!-- 正確 -->
#### 解法一：shadow 變數

說明文字
```

### 程式碼區塊

* 所有 ` ``` ` 區塊必須指定語言（MD040）
* 常用語言標籤：`go`、`bash`、`text`、`json`
* 程式碼區塊前後必須有空白行（MD031）

```markdown
<!-- 錯誤 -->
```
code here
```

<!-- 正確 -->

```go
code here
```

```

### 列表

* 使用 `*` 而非 `-` 作為項目符號（統一風格）
* 項目符號後只加一個空格（MD030）：`* 文字` 而非 `*   文字`

### 表格

* 欄位對齊使用 `: --- :` 語法
* 表格前後必須有空白行
* 中文括號後需要空格才接 `|`：`中文） |` 而非 `中文）|`（MD060）

## 程式碼範例規範

### 範例品質要求

1. **完整可執行**：範例要能直接複製執行，不缺 import 或 main 函數
2. **有輸出說明**：在程式碼下方或註解中說明執行結果
3. **對比寫法**：重要概念提供「不好的寫法」vs「好的寫法」對比
4. **真實場景**：範例使用接近實際開發的場景（User、Order、HTTP handler 等）

### 範例格式

```markdown
```go
package main

import "fmt"

func main() {
    // 說明這個範例展示什麼
    result := doSomething()
    fmt.Println(result)
}
```

```text
輸出：
result value
```
```

### 輸出說明

執行結果用獨立的 ` ```text ` 區塊呈現，標示「輸出：」：

```markdown
```text
輸出：
Hello, World!
```
```

## 內容撰寫風格

### 解釋方式

* 先說「是什麼」，再說「為什麼需要它」，最後「怎麼用」
* 用比喻幫助理解（例：Channel 像傳送帶，Goroutine 像工人）
* 指出常見陷阱（trap）並提供解法
* 複雜概念提供 ASCII 圖解

### 術語使用

* 中文術語後第一次出現時加英文對照：`閉包（Closure）`
* 保留關鍵字原文：`defer`、`goroutine`、`channel`（不翻譯）
* 函數名稱保留原文：`sync.WaitGroup`、`context.WithTimeout`

### 標注重要程度

* 危險用法加 `> **警告**：` 區塊
* 重要注意事項加 `> **注意**：` 區塊
* 最佳實踐加 `> **建議**：` 區塊

- [ ] 在 README.md 的「知識點速查表」加入對應查詢條目
